# Codex App かんたん切り替え 操作マニュアル

## このアプリでできること

ターミナルやコマンドを使わず、次の操作ができます。

- 通常のCodex GPTでCodex Appを使う
- 選択したOllama CloudモデルでCodex Appを使う
- 詳細設定からローカルモデルを選ぶ
- 推奨モデル`gpt-oss:120b-cloud`へすぐ戻す
- Ollamaモデルをインストール・確認する

## 事前準備

1. Codex Appをインストールします。
2. Ollama v0.24.0以降をインストールします。
3. Cloudモデルを使う場合はOllamaへサインインします。

## ダウンロード

最新版の配布ページ:

- [GitHub Releasesから最新版を選ぶ](https://github.com/aikiti/codex-app-easy-switcher/releases/latest)
- [Mac版（安定版v0.2.0）を直接ダウンロード](https://github.com/aikiti/codex-app-easy-switcher/releases/download/v0.2.0/Codex-App-Easy-Switcher-macOS.zip)
- [Windows版EXEを直接ダウンロード](https://github.com/aikiti/codex-app-easy-switcher/releases/download/v0.3.0/Codex-App-Easy-Switcher-Windows.exe)

## macOSでの起動方法

1. `Codex App かんたん切り替え.app`をダブルクリックします。
2. 初回起動時に警告が表示された場合、アプリを右クリックして「開く」を選びます。
3. 初回切り替え時にCodex Appを制御する許可を求められた場合は許可します。

## Windowsでの起動方法

1. `Codex-App-Easy-Switcher-Windows.exe`をダウンロードします。
2. EXEをダブルクリックします。コマンドプロンプトは表示されません。
3. SmartScreen警告が表示された場合、GitHubからダウンロードしたことを確認します。
4. 必要な場合だけ「詳細情報」→「実行」を選びます。

Windows版はコード署名証明書による署名を行っていません。GitHub以外から入手したEXEは
実行しないでください。Windows実機での初回検証では、起動・状態確認・モデル一覧確認を
先に行い、入力途中のCodexセッションがない状態で切り替えを試してください。

## 推奨モデルを使う

初期状態では、Ollama Cloudの`gpt-oss:120b-cloud`が選択されています。

1. 選択中のモデル名が`gpt-oss:120b-cloud`になっていることを確認します。
2. 「選択中のOllamaモデルで起動」を押します。
3. Codex Appが起動中の場合は警告を読み、「はい」を押します。
4. 切り替え後、Codex Appが起動します。

## Cloudモデルを変更する

1. メイン画面の「モデルを変更」を押します。
2. 「その他のCloudモデル」から使用するモデルを選びます。
3. 「選択したCloudモデルを使う」を押します。
4. メイン画面で選択中のモデル名と「Cloudモデル」を確認します。
5. 「選択中のOllamaモデルで起動」を押します。

モデルを選んだだけではCodex Appは再起動しません。

## 推奨モデルへ戻す

メイン画面の「推奨モデルに戻す」を押します。選択モデルが`gpt-oss:120b-cloud`へ戻ります。
実際に切り替えるには「選択中のOllamaモデルで起動」を押します。

## ローカルモデルを選ぶ

ローカルモデルは上級者向けです。Codex Appのファイル編集やエージェント機能が正常に動作しない
可能性があります。

1. 「モデルを変更」を押します。
2. 「詳細設定を表示」をオンにします。
3. ローカルモデル一覧から選びます。
4. 警告内容を確認して選択します。
5. メイン画面の「選択中のOllamaモデルで起動」を押します。
6. 起動前の警告を再度確認します。

問題が発生した場合は「推奨モデルに戻す」を押してください。

## モデル名を手動入力する

1. 「モデルを変更」を押し、「詳細設定を表示」をオンにします。
2. モデル名を入力し、「入力したモデルを選ぶ」を押します。
3. 未準備と表示された場合は「Ollama モデル管理」タブでインストールします。

安全のため、モデル名はOllama公式ライブラリで確認してください。

## 未準備モデルをインストールする

選択したモデルが一覧にない場合、起動は行われずモデル管理タブへ移動します。

1. 入力済みのモデル名を確認します。
2. 「モデルをインストール」を押します。
3. モデル名、種類、注意事項を確認して「はい」を押します。
4. 完了するまでPCを閉じずに待ちます。
5. メイン画面へ戻り、「選択中のOllamaモデルで起動」を押します。

ローカルモデルは数GB以上の通信量とディスク容量を使う場合があります。

## 通常のCodex GPTへ戻す

「通常のCodex GPTで起動」を押します。勉強会終了時や通常利用へ戻るときに使用します。

## ランチャー終了時の確認

Ollama接続状態でランチャーを閉じると、次から選べます。

- **通常のCodexに戻して終了**: 通常状態へ復元して終了します。
- **Ollama状態のまま終了**: 接続状態を維持して終了します。
- **キャンセル**: ランチャーへ戻ります。

## モデル一覧の見方

| 種類 | 説明 |
|---|---|
| Cloud | Ollamaのクラウド側で動作するモデル。Codex用として優先表示されます。 |
| ローカル | PCへダウンロードし、PC上で動作するモデル。Codex互換性は保証されません。 |

## 困ったとき

### 選択したモデルで起動できない

「Ollama モデル管理」タブでモデルが一覧にあるか確認します。Cloudモデルの場合はOllamaへの
サインイン状態とインターネット接続も確認してください。

### ローカルモデルでCodexが正常に動かない

「推奨モデルに戻す」を押し、`gpt-oss:120b-cloud`で起動してください。

### Ollamaが見つからない

画面の「Ollama公式ダウンロードページを開く」を押し、Ollamaをインストールしてください。

### unknown integration: codex-app と表示される

Ollamaをv0.24.0以降へ更新してください。

### 初回起動時にmacOSがアプリを開けない

アプリを右クリックして「開く」を選択してください。この配布版はApple公証を行っていません。

### Windows SmartScreenが表示される

Windows版EXEはコード署名証明書による署名を行っていないため、警告される場合があります。
GitHubの最新版Releaseからダウンロードしたことを確認し、必要に応じてRelease掲載のSHA256と
一致することを確認してから実行してください。

### WindowsでCodex Appを閉じられない

安全のため強制終了は行いません。Codex Appを手動で閉じてから、もう一度切り替えボタンを
押してください。

## 安全に使うための注意

- 接続切り替え前に、Codex Appの入力途中の内容を確認してください。
- ローカルモデルのインストール前に、通信量と空きディスク容量を確認してください。
- モデル名はOllama公式ライブラリから確認してください。
- 業務上の機密情報をCloudモデルへ送信する場合は、所属組織のルールに従ってください。
